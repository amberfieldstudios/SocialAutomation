# @social/ui — dashboard

A React + Vite + TypeScript dashboard over `@social/api`. Views: **Accounts**
(connected-accounts manager, reconnect flow), **Composer** (one description in,
live per-platform previews out), **Queue & Schedule** (upcoming/in-flight/failed/
dead-lettered jobs + schedules), **History** (per-post publish results), and
**Analytics** (per-campaign metric rollups, colorblind-safe).

No real platform or AI credentials exist anywhere in this project. Composer
previews are generated by `@social/ai`'s deterministic `MockProvider` and
validated with each connector's real `validatePost`; nothing is published
until "Submit campaign" is pressed, and even then jobs fail against the real
Discord/Twitch/Bluesky APIs without real OAuth tokens (by design — see
`packages/api/src/seed.ts`).

## Run it (dev)

From the repo root:

```bash
# 1. Seed a dev SQLite DB (accounts, two real campaign submissions, dead-lettered
#    jobs from the missing credentials, plus synthetic "already published" history/analytics).
pnpm --filter @social/api run seed

# 2. Start the API (Fastify, http://127.0.0.1:4000) against that same DB file.
SOCIAL_DB_FILE=./packages/api/dev.sqlite pnpm --filter @social/api run dev

# 3. In another terminal, start the UI dev server (http://127.0.0.1:5173,
#    proxies /api to the server above — see packages/ui/vite.config.ts).
pnpm --filter @social/ui run dev
```

Open http://127.0.0.1:5173.
